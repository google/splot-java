Class Overview

A nested persistent state manager that aggregates the persistent state of several objects under a single persistent state interface which can itself be managed by another PersistentStateManager.

Note that initWithPersistentState(Map) must be called before any objects are managed via startManaging(String, PersistentStateInterface). The behavior of calling initWithPersistentState(Map) after the first call to startManaging(String, PersistentStateInterface) is undefined.

public Map<String, Object> copyPersistentState ()

Retrieves a copy of the state for the instance that should be saved to nonvolatile storage. When reconstructing the associated instance, call initWithPersistentState(Map) to restore this state.

The data contained in the returned Map should be considered opaque unless specified otherwise by the implementing technology.

The types for the values contained in the map are only guaranteed to be faithfully restoreable if they are one of the following types:

  • Integer
  • Double
  • Float
  • byte[]
  • String
  • Map<String,Object>, where Object is also one of these acceptable types
  • java.util.List<Object>, where Object is also one of these acceptable types
This allows the object to be easily serialized to CBOR.

This method MUST be thread safe.

Returns
  • the persistent state of this instance

public void initWithPersistentState (Map<String, Object> persistentState)

Restores a persistent state previously retrieved from copyPersistentState(). This should be performed only once when this instance is being initialized. The behavior for calling this method more than once is undefined.

Parameters

public void refresh (String objectName)

Reads the current persistent state of the indicated managed object and writes it to non-volatile memory. Note that flush() should be called after calling this method if the intent is to save the state before powering down.

public void refresh ()

Reads the current persistent state of all managed objects and writes it to non-volatile memory. Note that flush() should be called after calling this method if the intent is to save the state before powering down.

public void reset ()

Resets all persistent states managed by this object. Note that any objects which are currently being managed will end up restoring their state if they end up indicating that they need to be saved or if refresh() is called.

This method is generally used to implement factory-reset capabilities.

public void reset (String objectName)

Resets the persistent state for a particular object. Note that if the given object is currently being managed, then the state will be restored if they end up indicating that they need to be saved or if refresh(String) is called.

Parameters
objectName the name of the object to erase the persistent state

public void setPersistentStateListener (PersistentStateListener listener)

Sets the listener to be called when the persistent state needs to be stored to non-volatile memory. There can only be one listener.

This method MUST be thread safe.

Parameters
listener the listener to use to indicate that this instance needs to save the persistent state to non-volatile memory.

public synchronized void startManaging (String objectName, PersistentStateInterface objectInstance)

Starts managing the persistent state of the given object. If there exists state for this objectName that was previously saved to non-volatile memory, it is restored by calling initWithPersistentState(Map). Then the given instance is monitored for indications that it needs to refresh the persistent state to non-volatile memory.

Parameters
objectName the name of the object being managed
objectInstance the instance of the object being managed

public synchronized void stopManaging (String objectName)

Stops the manager from monitoring an object previously registered with startManaging(String, PersistentStateInterface). If the given object is not being currently managed, calling this method does nothing.

Parameters
objectName the name of the object to stop managing
