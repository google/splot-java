Class Overview

Abstract class for more easily implementing local FunctionalEndpoints. This class is intended by be subclassed by developers who are implementing FunctionalEndpoints which are locally hosted---meaning those functional endpoints which will be used to manipulate aspects of the device this code is running on. Subclasses of this class should NOT be used to implement non-local FunctionalEndpoints: in those cases a class directly implementing the methods of FunctionalEndpoint should be used.

Note that there are special subclasses of this class which provide native support for scenes and transitions: LocalSceneFunctionalEndpoint and LocalTransitioningFunctionalEndpoint, respectively. If your functional endpoint needs to support either, you should subclass one of those classes instead.

public ListenableFuture<?> addValueToProperty (PropertyKey<T[]> key, T value)

Asynchronously adds the given value to a list/set property

The caller can optionally block until completion by calling the get() method, or can be notified of completion by adding a listener via addListener(Runnable, Executor).

Note that the order of the resulting value is specified by the property itself (if at all).

The returned future may throw one of the following checked exceptions (as the cause to a java.util.concurrent.ExecutionException) when completed:

  • PropertyException, or one of its subclasses:
    • PropertyReadOnlyException if the property is read-only (add value requires read/write)
    • PropertyWriteOnlyException if the property is write-only (add value requires read/write)
    • PropertyNotFoundException if the property isn't present on this fe
    • PropertyOperationUnsupportedException if add value is not allowed on this property
    • InvalidPropertyValueException if the value being added was rejected
  • TechnologyException if there was a technology-specific problem

Parameters
key the key identifying the property and its type
value the value to be added to the list or set
Returns
  • a future that can be cancelled or monitored asynchronously for completion

public ListenableFuture<?> applyProperties (Map<String, Object> properties)

Asynchronously change the value of multiple properties at once.

To add properties to a Map in a type-safe way, use the putInMap(Map, T) method.

The caller can optionally block until completion by calling the get() method, or can be notified of completion by adding a listener via addListener(Runnable, Executor).

The applied properties must all be from the same section. The behavior of passing properties from different sections is undefined.

The returned future may throw one of the following checked exceptions (as the cause to a java.util.concurrent.ExecutionException) when completed:

  • PropertyException, or one of its subclasses:
    • PropertyWriteOnlyException
    • PropertyNotFoundException
    • PropertyOperationUnsupportedException
    • InvalidPropertyValueException
  • TechnologyException if there was a technology-specific problem
However, it this functional endpoint is a group, then this method's future will generally not throw PropertyException for state section properties.

Parameters
properties a map keyed by the property name containing the new values for those properties.
Returns
  • a future that can be used to block execution until the action is complete, cancel the action, or monitor the action asynchronously for completion

public Map<String, Object> copyPersistentState ()

Retrieves a copy of the state for the instance that should be saved to nonvolatile storage. When reconstructing the associated instance, call initWithPersistentState(Map) to restore this state.

The data contained in the returned Map should be considered opaque unless specified otherwise by the implementing technology.

The types for the values contained in the map are only guaranteed to be faithfully restoreable if they are one of the following types:

  • Integer
  • Double
  • Float
  • byte[]
  • String
  • Map<String,Object>, where Object is also one of these acceptable types
  • java.util.List<Object>, where Object is also one of these acceptable types
This allows the object to be easily serialized to CBOR.

This method MUST be thread safe.

Returns
  • the persistent state of this instance

Deletes this Functional Endpoint, if possible. Note that not all functional endpoints can be deleted. If the delete operation cannot be performed, the future will indicate an exception. Returns
  • a future object returning a boolean. If the boolean is true, the object was successfully deleted. If the boolean is false, the object is not deletable.

public T getCachedProperty (PropertyKey<T> key)

Synchronously return the most recent cached value of a specific property.

Note that if this is a "local" Functional Endpoint (isLocal() returns true ), then the returned values are the actual values of the properties instead of cached values.

Note that this method offers no way to differentiate the following cases where null is returned:

  • The property hasn't been cached yet.
  • The property isn't supported by this functional endpoint.
  • The value of the property is actually null.
If such ambiguity is unacceptable, you will need to either use the fetchProperty(PropertyKey) method or inspect the maps returned from copyCachedState(), copyCachedConfig(), and/or copyCachedMetadata().

Parameters
key the key for the property to fetch
Returns
  • the cached value of the property, or null if the property is not cached.
